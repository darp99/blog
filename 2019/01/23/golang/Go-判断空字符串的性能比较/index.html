Go: 判断空字符串的性能比较

Jan 23 2019 Golang

字符串在 Go 内部中是一个结构体，一个字段记录长度，一个字段记录指针位置。

判断字符串为空最常见的是 data == "" 这样，之前我认为这里分配了一个空字符串会占用分配的时间，所以我把一些判断都改成了 len(data) == 0 这样就不用分配一个空字符串的空间了，效率应该会提升。

但最近我在 Go 源码中看到很多使用第一种判断是否为空的写法，这让我有点怀疑是否我的优化是否正确。
